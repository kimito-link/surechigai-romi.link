## 背景

surechigai-romi.link の物語LP（`public/lp/`）を、日本でバズった2つのFable5製LP
（①SAKURA=墨絵桜の静止画KV・16万インプ ②MINAMO=コードで描く波紋の水面・29万インプ）を
超える最高の物語LPにする。芭蕉「月日は百代の過客」の四季道中記・和紙墨明朝の世界観は死守。

**直近で失敗した実装（1df9bb4bc）を作り直す**のが最優先。結章「湖に、月が、逆さに浮かんでゐる」に
Canvas水面を入れたが、月光が巨大な縦の柱になり逆さ月の球体が見えず、position:fixedとスクロール
文の干渉で月が見切れた（本番でユーザーが「ひどい」と評価）。

## §0. 会議で確定した「失敗の真因」と「正しい技術解」（最重要）

| 論点 | 前回失敗（1df9bb4bc） | 正しい解（会議で確定・実装必須） |
|---|---|---|
| **レイヤー** | `position:fixed` 全画面Canvas。ビューポート基準で描画し、スクロールで本文と座標系がずれ、月が上端で見切れた | **結章`<section>`内に `position:absolute` のCanvas**を置く。本文テキストは同一スタックコンテキストで`z-index`を上げて前面に。Canvasの座標系が本文と一致し、スクロール量に依存しない |
| **柱状月の原因** | 逆さ月を横スライスに切り、各スライスを横に`wob`ずらし。**縦方向の情報が失われ**、円が上下に伸びて光の帯（柱）になった | **月ディスクを完全な円でオフスクリーンバッファに描く → 水面の高さ場(height field)の勾配/法線でピクセル単位にサンプリング変位**。円形を保ったまま屈折で揺らす |
| **波紋の物理** | `{x,y,r,a}`の波紋リストをsin/cosで毎ピクセル評価。O(N波紋×Mピクセル)でモバイル60fps不可、揺れも荒い | **2バッファ Height-Field（離散波動方程式）**。O(W×H)固定。低解像度バッファ(例128〜256px)で計算し`drawImage`で拡大。GPU不要で60fps |
| **配置基準** | fixedのtop/left固定で見切れ | **`section.getBoundingClientRect()`でビューポート内の位置を毎フレーム取得**し、Canvasローカル座標へ変換。どのスクロール量でも画面内に安定 |

**この4点が作り直しの核。特に「月ディスク→高さ場サンプリング変位」と「2バッファHeight-Field」は
前回失敗の直接の解なので、擬似コード（§2）通りに実装すること。**

## §1. 全体コンセプト

**「読むと、世界が応える LP」** — 芭蕉の句を読み進める（スクロールする）と、四季の情景が
文に応じて息づき、要所（KV・結章）では**ユーザーが触れると世界が反応する**（波紋・墨）。
SAKURAの静止画・MINAMOの水面単体を超えるのは「物語＋インタラクション＋静謐」の三位一体。
派手さで勝つのではなく、**芭蕉の"間"の中に、触れられる一点を置く**ことでインパクトと品を両立する。

## §2. 失敗した水面の作り直し（最優先・最も具体的に）

### レイヤー構成（HTML/CSS）

```
<section id="lake-scene" class="story ...">   /* position:relative; スタックコンテキスト生成 */
  <canvas id="lakeCanvas"></canvas>            /* position:absolute; inset:0; z-index:1（水面描画専用） */
  <div class="veil"></div>                     /* 既存の暗幕 */
  <p class="ph" data-mizu="1">湖に、月が、逆さに浮かんでゐる。</p>  /* position:relative; z-index:2（本文は前面） */
  ... 結章の他の文 ...
</section>
```
- `#lake-scene { position:relative; }` が肝。Canvasはこのsection内で`absolute; inset:0`。
- **`position:fixed`は使わない。** section内absoluteなら、Canvasはsectionと一緒にスクロールし、
  本文と座標系が常に一致する。
- Canvasの実ピクセルは `clientWidth*dpr × clientHeight*dpr`、CSSで`width:100%;height:100%`。
  `ResizeObserver`でサイズ変化時に再設定。

### 水面の height field（2バッファ波動方程式）

低解像度の高さ場バッファ2枚（`prev`, `cur`）を交互更新。各セルは離散波動方程式：

```
// グリッド解像度（軽さ優先。実Canvasより粗くてよい。例 GW×GH = 160×90）
cur[i] = (prev[i-1]+prev[i+1]+prev[i-GW]+prev[i+GW])/2 - cur[i]   // 波の伝播
cur[i] *= damping                                                 // 0.96〜0.99 で減衰
// 毎フレーム prev と cur を swap
```
- クリック/タッチ時、その点のセルに高い値を注入（波の発生源）。
- 常時のさざなみ：ランダムな微小セルに毎フレーム弱い注入、または低振幅のノイズ。
- **計算はグリッド解像度分だけ（O(GW×GH)）**。160×90=14400セルなら60fps余裕。

### 逆さ月を「球体」に描く（柱化を回避）

```
// 1) 月ディスクをオフスクリーンに完全な円で描く（一度だけ・キャッシュ）
moonBuf = offscreen(2R×2R)
radialGradient(中心#fdf6e3 → 縁#d9cba0) で塗った円

// 2) 水面領域の各ピクセル(x,y)について、height fieldの勾配で「どこを映すか」をずらす
for each pixel (x,y) in water region:
    // 高さ場の勾配（隣接セル差分）＝水面の傾き＝屈折方向
    dx = height(x+1,y) - height(x-1,y)
    dy = height(x,y+1) - height(x,y-1)
    // 鏡像元の座標（月ディスク内のどの点を映すか）。屈折分ずらす。
    srcX = (x - moonReflX) + dx * refractStrength
    srcY = (reflectAcross(y) ) + dy * refractStrength   // 水面境界で上下反転した位置
    サンプル = moonBuf のその座標の色
    水面ピクセルに合成（水の暗さ・青みを乗算）
```
- **ポイント**：月は常に「完全な円のバッファ」から取る。ずらすのは"どこを映すか"だけ。
  だから円が伸びて柱にならない。波が来た所だけ屈折で歪む＝水面で月が揺れて見える。
- 実装簡略版：ピクセル単位が重ければ、月ディスクを`drawImage`で逆さ配置し、
  height fieldを使った**行ごとの横シフト＋縦の微小シフト**で近似してもよい（ただし前回の
  「横スライスのみ」は禁止＝縦も動かすこと）。まずは擬似コードの正攻法を試す。

### 月・逆さ月の配置

```
rect = lakeSection.getBoundingClientRect()
// 空の月：section上部から一定割合。逆さ月：水面境界の下に鏡像。
moonX = W*0.68
moonY = W*0.16   // 幅基準にすると縦横比で破綻しにくい
horizon = H*0.5  // 水面境界（section高さ基準）
moonReflY = horizon + (horizon - moonY)   // 鏡像
```
- 結章に入った瞬間、月が画面上1/4、水面が中〜下、逆さ月が水面直下に来る理想像。
- reduced-motion時はCanvasを起動せず、既存 `tsuki-mizu.png` 静止画にフォールバック（現行の作法を踏襲）。

### 起動/停止
- 現行の`data-mizu`検知（`updateNarrative`内）を流用。結章が画面に入ったら`start()`、
  離れたら`stop()`（rAF停止で省電力）。既に配線済みなのでロジックだけ差し替える。

## §3. KV（ファーストビュー）— SAKURA超え（P1・水面の後）

会議案「墨と水の対話」。ただし**まず水面（§2）を成功させてから**着手。
- ファーストビューに、芭蕉の発句を**墨で書かれていく**演出（スクロール/時間で一画ずつ描画、
  筆速で線幅変化）。背景はSVG和紙テクスチャ（軽量）。
- 静止画SDに頼らず「読む/触れると応える」体験で、SAKURAの静止KVを超える。
- reduced-motionでは完成した句を静止表示。

## §4. SD画像の扱い

- 全置換は不要。**要所だけコード描画に置換**：結章の月・水面（§2で置換）、KVの墨（§3）。
- 四季の情景写真（雪国・桜・滝・祭・紅葉）はSD維持。ただし**低解像度のボケが目立つものは
  差し替え候補**。判断基準：全画面背景で使われ、ボケが目立つもの＞小さく添える程度のもの。
- 当面はコード描画部分（月・墨）の品質で"見せ場"を作り、SD写真は世界観の下地に徹させる。

## §5. 実装フェーズ（小さく安全に）

1. **Phase 1（最優先）: 水面の作り直し**。現行の失敗Canvas（app.jsのmizuエンジン）を、
   §2のheight field方式に置換。まず「月ディスク→高さ場サンプリング」で球体の逆さ月が
   静止状態で正しく見えることを確認 → 次に波紋 → 次にsection内absolute配置で見切れ解消を確認。
   各段階でPlaywright（reducedMotion:"no-preference"）＋実機で検証。**app.js?v=NN を必ず上げる**。
2. **Phase 2: reduced-motion/モバイル**。静止フォールバック、横はみ出し無し、60fps計測。
3. **Phase 3（P1）: KVの墨演出**（§3）。水面が完成してから。
4. **Phase 4: SD写真の品質棚卸し**（§4）。差し替え候補の洗い出しは提案に留め、差し替えは要確認。

## §6. 検証

- Playwrightは**`reducedMotion:"no-preference"`を必ず渡す**（渡さないとreduce扱いで水面が起動しない＝前回ハマった罠）。
- 結章へは`document.querySelector("[data-mizu]").scrollIntoView({block:"center"})`後、rAFが数フレーム
  回るのを待ってからスクショ。月・逆さ月が画面内に収まり、柱でなく球体で、波紋が広がることを目視。
- 実機（スマホ）で最終確認。CDNキャッシュのため`app.js?v=NN`を上げてデプロイ。

## §7. 死守・禁止（地雷マップ）

- 世界観死守：和紙/墨/明朝/旧字体（ゐ・思ひ出）/四季道中記/「会いたい君がいる現在地」。
- 既存資産を壊さない：サウンドノベルのスクロール送り、四季BGM、効果音、キャラりんく、
  フィナーレのアプリ紹介章「巻・携」。
- 技術：素のHTML/CSS/JS維持。重いライブラリ（Three.js等）追加しない（Canvas 2Dで足りる）。
- **`position:fixed`全画面Canvasは二度と使わない**（前回失敗の主因）。
- app.jsを変えたら`index.html`の`app.js?v=NN`を必ず+1（CDN immutableキャッシュ対策）。
