**重要な教訓（次モデルも厳守）**:
- アイコンJS等の**動的importでReactコンポーネントを直接描画して常時マウントするパターンは禁止**（`lazy()`+Suspenseは可）。
- **デプロイ確認は`version.json`一致だけでは不十分**な場合がある。Metroの子チャンクは親チャンクのファイル名に参照先を含めないため、CDN/ブラウザキャッシュに旧チャンクが残り得る。`scripts/bust-entry-cache.cjs`がHTML内の全チャンク参照に`?v=<sha>-<ts>`を付与する対策済みなので、通常デプロイなら問題ないはず。疑わしい時はチャンク連鎖をcurlで辿って確認する。
- React Queryのmutation/queryの「結果オブジェクト」をuseCallback/useEffectの依存に入れない（安定参照でないため無限ループの原因になる）。
- Vercel/Cloudflare経由のクライアントIP判定は`cf-connecting-ip`を最優先にする（`x-forwarded-for`だけだと複数プロキシで不安定）。

## 4. 未着手のタスク（次のアクション候補）

### 4-1. DB接続プールとRailwayプランの整合確認（Stop And Ask）

`server/db/connection.ts`の`max: 2`（postgres.js接続プール）は妥当な初期値だが、
Vercel Functionの同時実行数が増えるとRailway側の総接続数上限に達する可能性がある。
**Railwayの現在のプラン・同時接続数上限を確認してから`max`値の調整要否を判断すること**（ユーザー確認必須、DB課金プランに影響しうる）。

### 4-2. 認証込みのチェックイン実地確認（未実施のまま）

`.auth/auth-state.json`が無いか空の状態。`pnpm e2e:auth-save`でXログインを完了させれば
以後の無人検証が可能になる（過去に偽陽性で失敗していたが該当バグは`fbfd366`で修正済みのはず）。
実際にログイン済み状態でチェックイン→保存の一連の動作を人間の目で見た確認はまだ済んでいない。

### 4-3. ピンチズーム地図体験（ユーザー要望、設計未着手）

iPhone写真アプリのような「拡大すると位置精度が上がる」体験の要望あり。

- 現状の地図実装（`components/organisms/precision-tile-map.tsx`）は+/-ボタンのみで、
  ピンチジェスチャー・パンドラッグの実装が一切ない。
- `react-native-gesture-handler`(~2.28.0)・`react-native-reanimated`(~4.1.6)は**既に依存に入っている**ので
  新規ライブラリ追加なしで実装できる。
- 座標変換の純関数（`lib/map/tile-geo.ts`の`pixelToLatLng`/`latLngToWorldPixel`）は既にあるので、
  ジェスチャー→ズーム値・中心座標の更新ロジックを追加するだけで済む設計。
- ユーザーに対象画面を確認したところ「Fableで最高の体験を考えて」という回答だったため、
  次回セッションで対象画面（公開共有ページ/自分の軌跡/図鑑/チェックインのいずれか、または複数）を
  決めてから設計するとよい。会議ハーネス→Fable設計の3段構えプロセスを推奨。

### 4-4. グルメマップ・思い出機能（ユーザー要望、設計未着手・規模大）

食べた物の写真＋店舗情報を地図に記録したいという要望。ユーザーが送った写真（そば・ラーメン等）から
イメージするに、「チェックイン時に写真を添付できる」「地図上で写真サムネイルがピンになる」体験。

- 現状、`locations`テーブルに写真・グルメ関連のカラムは一切ない（**新規テーブルが必要＝DBスキーマ変更**）。
- **DBスキーマ変更を伴う規模の大きい新機能**なので、着手する場合は先に会議ハーネス→Fable設計の
  3段構えプロセスを踏むことを推奨。

## 5. 事業設計ドラフト（要点、詳細は過去のチャット履歴参照）

ユーザーから「事業責任者AI」として20項目の事業設計を依頼された経緯がある。技術部分は具体的に回答し、
価格・法人展開・特許商標は「たたき台」として提示し断定を避けた。要点:

- **事業定義**: 「移動ログ・コレクションアプリ」というカテゴリ。位置情報アプリと名乗らない。
- **狙うべき層の優先順位**: 1.推し活・配信者クラスタ 2.ドライブ/車中泊層 3.旅行/スタンプラリー層 4.一般層
- **課金モデルの設計思想**: 「見る・繋がる・シェアする」は無料無制限、「貯める機能の高度活用」を課金対象に。
  記録を消させる無料プランは核となる価値提案を壊すので避ける。
- **価格ドラフト**: 無料0円 / ライト300-400円 / プレミアム900-980円 / 法人は案件ごと見積+月額固定。
  **未検証、市場テストを経てから確定すべき**と明記済み。
- **法人向け機能**（キャンペーン管理・スタンプラリー等）は現状DBスキーマに存在しない。新規テーブルが必要。
- **KPI**: 「ユーザー数の増加とAPIリクエスト数の増加が比例しているか」が人気/バグの判別軸。
- Vercel異常アクセス（Edge Requests急増）の主因は#2（presence.pulse無限ループ）だった可能性が高いと判定済み。
- 収支の具体的な数字・特許商標の断定的助言は意図的に避けた（未検証の仮定や専門外領域のため）。
