**重要な教訓（次モデルも厳守）**:
- アイコンJS等の**動的importでReactコンポーネントを直接描画して常時マウントするパターンは禁止**（`lazy()`+Suspenseは可）。
- **デプロイ確認は`version.json`一致だけでは不十分**な場合がある。Metroの子チャンクは親チャンクのファイル名に参照先を含めないため、CDN/ブラウザキャッシュに旧チャンクが残り得る。`scripts/bust-entry-cache.cjs`がHTML内の全チャンク参照に`?v=<sha>-<ts>`を付与する対策済みなので、通常デプロイなら問題ないはず。疑わしい時はチャンク連鎖をcurlで辿って確認する。
- React Queryのmutation/queryの「結果オブジェクト」をuseCallback/useEffectの依存に入れない（安定参照でないため無限ループの原因になる）。
- Vercel/Cloudflare経由のクライアントIP判定は`cf-connecting-ip`を最優先にする（`x-forwarded-for`だけだと複数プロキシで不安定）。
